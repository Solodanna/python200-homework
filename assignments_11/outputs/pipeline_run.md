# Week 11 Pipeline Run Reflection

My pipeline did not run cleanly on the first try because the transform task failed when my OpenAI API key in .env was still a placeholder value. After replacing it with a valid key and confirming az login was active, the second run completed successfully from extract through load. In the Prefect UI, I could see each task transition by state, and the run graph made it clear that transform was the failed step on the first attempt. I did not see retries during the successful run, but the extract task is configured for retries and would retry automatically on transient API failures. If I were deploying this on a daily schedule, I would add an automated notification policy so failed runs send an alert with the failed task name and exception summary.
